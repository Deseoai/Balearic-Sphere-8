# Wireframes + Userflow (MVP)

## Hauptscreens

1. Landing
2. Request Access Form
3. Application Pending Status
4. Member Home (Network Lite)
5. Profile
6. Credits Wallet
7. AI Concierge
8. Marketplace Feed
9. Marketplace Listing Detail
10. Forum Rooms
11. Forum Thread
12. Notifications
13. Access Level + Eligibility
14. Admin Review Queue
15. Admin Application Detail

## Primarer Userflow

1. Visitor landet auf Hero
2. Klick auf `Request Access`
3. Bewerbung absenden
4. Pending Status + Trust Microcopy
5. Review abgeschlossen
6. Zugang + initial Credits
7. Erste AI Anfrage
8. Node/Profile Unlock
9. Intro/Listing/Forum Interaktion

## Gate-Logik

- Jede sensible Aktion prueft:
  - Rolle
  - Access Level
  - Credit Kosten
  - Verification Pflicht

## UX-Regeln

- Kein Social-Feed als Start
- Start im Network-Kontext
- Bewusst limitierte Sichtbarkeit (Shadow Nodes)
- Upgrade-Hinweise durch Nutzen, nicht durch Druck
